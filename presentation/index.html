Foundations and Basics

Erlang

What is the big deal?!

* Created by Swedish Telcom co Erikson in 1986 * Goals: high concurrency, fault tollerance, and scalability * Language and a RunTime * Functional and Dynamic with a Prolog inspiration * Ugly and spikey * Used @ ASY on 2 projects

Erlang

OK, that is a big deal...

But how does it do it?

* Actor model + message passing + no share state * Immutable data structures (single asignment, list transformation) * Cheap and fast "process" creation * OTP - common patterns and supervision/monitoring * Fail Fast = cut out the hurt part * Applications + Nodes + Releases + Hot codeswap = UPTIME * "9 nines" of uptime on Erikson switch

Elixir

but, why?

* Jose wanted the benefits but with a lower cost * Ruby inspired * Better tooling with more developer focus
